CAMEL HEALTH MODEL — already wired in and working
==================================================

This is DONE — no setup needed. The trained model is already here and
connected to the app. Open the Camel Farm page → "AI Health Diagnosis",
take/upload a photo, tap Run Diagnosis.

What's here:
  camel_triage_model.onnx   ← the trained model (YOLOv8 classification, exported to ONNX)
  labels.json               ← class order: High_Risk, Low_Risk, Medium_Risk

How it runs:
  - Inference runs in the browser via ONNX Runtime Web (bundled in /vendor,
    so it works OFFLINE — no internet needed after first load).
  - Input: the photo, resized to 224x224. Output: one of three RISK TIERS.
  - Code: src/utils/camelModel.js  (mirrors palmModel.js)
  - Routing: src/utils/diagnosis.js sends type="camel" here.

About the model (for the writeup / honest framing):
  - It reports a RISK TIER (how urgent: Low / Medium / High), NOT a specific
    disease. The dataset was too small to name individual diseases reliably,
    so the 7 conditions were grouped into 3 urgency tiers.
  - Validated at ~83% accuracy on a held-out test set.
  - Safe failure pattern: in testing, no truly High-risk case was ever
    misclassified as Low — all errors fell to an adjacent tier.
  - It is a screening aid for a vet to confirm, NOT a diagnosis.
